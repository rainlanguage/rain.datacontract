newContainer

Prepares a container ready to write exactly length_ bytes at the returned pointer_. The caller MUST write exactly the number of bytes that it asks for at the pointer otherwise memory WILL be corrupted.

function newContainer(uint256 length_)
    internal
    pure
    returns (DataContractMemoryContainer container_, Pointer pointer_);

Parameters

NameTypeDescription
length_uint256Caller specifies the number of bytes to allocate for the data it wants to write. The actual size of the container in memory will be larger than this due to the contract creation prefix and the padding potentially required to align the memory allocation.

Returns

NameTypeDescription
container_DataContractMemoryContainerThe pointer to the start of the container that can be deployed as an onchain contract. Caller can pass this back to write to have the data contract deployed (after it copies its data to the pointer).
pointer_PointerThe caller can copy its data at the pointer without any additional allocations or Solidity type wrangling.

write

Given a container prepared by newContainer and populated with bytes by the caller, deploy to a new onchain contract and return the contract address.

function write(DataContractMemoryContainer container_) internal returns (address);

Parameters

NameTypeDescription
container_DataContractMemoryContainerThe container full of data to deploy as an onchain data contract.

Returns

NameTypeDescription
<none>addressThe newly deployed contract containing the data in the container.

read

Reads data back from a previously deployed container. Almost verbatim Solidity docs. https://docs.soliditylang.org/en/v0.8.17/assembly.html#example Notable difference is that we skip the first byte when we read as it is a 0x00 prefix injected by containers on deploy.

function read(address pointer_) internal view returns (bytes memory data_);

Parameters

NameTypeDescription
pointer_addressThe address of the data contract to read from. MUST have a leading byte that can be safely ignored.

Returns

NameTypeDescription
data_bytesThe data read from the data contract. First byte is skipped and contract is read completely to the end.

readSlice

Hybrid of address-only read, SSTORE2 read and Solidity docs. Unlike SSTORE2, reading past the end of the data contract WILL REVERT.

function readSlice(address pointer_, uint16 start_, uint16 length_) internal view returns (bytes memory data_);

Parameters

NameTypeDescription
pointer_addressAs per read.
start_uint16Starting offset for reads from the data contract.
length_uint16Number of bytes to read.
